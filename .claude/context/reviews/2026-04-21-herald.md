## Process Note

The GraphQL mutation `updateProjectV2Field(input: { singleSelectOptions: [...] })` regenerates option IDs even when names are identical, which wipes existing field-value assignments on all project items. The project-management skill's `phase-management.md` reference documents this risk for CLI-based field deletion, but the same warning applies to the GraphQL path. Future phase-option additions should either be performed via the GitHub web UI (which preserves assignments) or scripted as "mutate → immediately restore" as was done here.
